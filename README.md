#数据结构

##用户表

> 1. id
> 2. name 登陆名
> 3. nickname  显示名
> 3. password   密码
> 2. email  邮箱地址
> 4. group  用户组
> 5. create_at  创建时间
> 6. update_at  更新时间
> 6. status    状态

## 用户组表

> 1. id
> 2. name  组名
> 3. privileges  允许的权限,可以序列化格式存储 直接存储url(模块的地址)

## 模块表
> 1. id
> 2. name 模块名
> 3. parent 父模块
> 3. url  模块url地址
